本日志记录了7月19日的工程实践内容——使用热风返修台完成第一块电驱板的SMT贴片手焊（U1 TMI8720PD、Q1/Q2 AGM210MAP）和插件元件焊接，并讨论确定了双电机驱动方案（放弃并联驱动，采用双板独立驱动方案）。日志涵盖手工焊接工艺与系统架构决策。

一、今日工作内容

1.1 PCB焊接

SMT贴片手焊：

  • 对 U1（TMI8720PD，DFN-10-EP）和 Q1/Q2（AGM210MAP，PDFN-8）进行热风手焊
  • 使用热风返修台 300~350°C，焊锡膏辅助
  • 注意芯片摆放方向和引脚对齐，焊后目视检查 EP 焊盘吃锡情况

插件元件焊接：

  • 焊接 F1（PPTC 保险丝 H60-250）、C1/C2/C3（电解电容）、R1/R2（贴片电阻）
  • U2/U3（MKDS 接线端子）、U4（1×6P 排针）
  • 特别注意电解电容正负极方向（长脚为正，壳体带灰色条为负）
  • 保险丝、端子焊牢，保证机械强度和过流能力

1.2 双电机驱动方案讨论

方案描述结论
方案A：并联驱动 两个后轮电机并联到同一块板的 U3 端子 ❌ 放弃 — 两轮强制同速，无法满足差速转向需求
方案B：双板独立驱动 两块电驱板各驱动一个后轮电机，独立调速 ✅ 采用 — 可控性强，支持差速和原地转向

ESP32 GPIO25 ──→ 板1 IN1 ──→ 左轮电机 ESP32 GPIO26 ──→ 板1 IN2 ESP32 GPIO27 ──→ 板2 IN1 ──→ 右轮电机 ESP32 GPIO28 ──→ 板2 IN2 ESP32 GND ──→ 板1 U4-3 ──→ 板2 U4-3（共地） 12V 母线 ──→ 板1 U2-1 ──→ 板2 U2-1

二、关键技术点

2.1 DFN/PDFN 热风手焊工艺

步骤操作要点
1焊盘涂锡膏薄涂均匀，避免连焊
2摆放芯片对齐引脚和焊盘，注意方向标记
3热风焊接300~350°C 中速吹焊，焊锡熔化后移开
4目视检查EP焊盘吃锡情况，引脚是否连焊

2.2 双板驱动架构

方案B 核心优势：

  • 左右轮独立调速，支持差速转向
  • 可实现原地转向（两轮反向转动）
  • 单板故障时另一板仍可工作
  • 接线清晰，调试方便

三、工程伦理与设计反思

🛠 工艺：热风手焊的风险管控

DFN 封装手焊是本次实习中工艺难度最高的环节——引脚在芯片底部不可见，温度过高可能损坏芯片，锡膏过多可能导致连焊。通过控制温度 300~350°C、薄涂锡膏、焊后目视检查 EP 焊盘吃锡情况，最大限度降低手焊风险。

🔌 安全：电解电容极性验证

电解电容接反会爆裂——这不是危言耸听。焊接前用万用表确认极性（长脚为正、壳体灰色条为负），焊接后再次目视检查。一个简单的极性检查动作，可以避免整块板子报废甚至安全事故。

⚙ 系统：双板方案的工程权衡

并联驱动（方案A）省一块板子但牺牲了差速能力——对于一辆需要灵活转向的卡丁车来说，这是不可接受的。双板方案（方案B）虽然增加了物料成本和接线复杂度，但换来了可控性、可靠性和扩展性。工程决策不是选"最便宜的"，而是选"最合适的"。

📋 规范：焊接工艺的可复现性

第一块板的焊接过程记录了详细的工艺参数（温度、步骤、检查要点），确保第二块板的焊接质量与第一块一致。焊接不是"焊上去就行"，而是要有标准作业流程来保证可复现性。

四、遇到的问题与解决方案

⚠️
问题1：DFN 封装芯片引脚在底部，难以观察焊接质量

TMI8720PD 为 DFN-10-EP 封装，引脚在芯片底部，焊接时无法直接看到焊锡是否熔化流动。

✅
解决方案：采用"先涂锡膏再放芯片"的工艺顺序——在焊盘上薄涂一层焊锡膏后放置芯片，热风吹焊时锡膏熔化会从芯片侧面溢出，通过观察侧面溢锡情况判断焊接质量。
教训：DFN 焊接的关键是锡膏用量控制——太少焊不牢，太多会连焊。
⚠️
问题2：并联驱动方案无法满足差速需求

两个电机并联到同一块板，转速被强制同步，转向时内外轮无法产生速度差，影响操控性。

✅
解决方案：采用双板独立驱动方案，每块板独立控制一个电机。左右轮独立调速，支持差速转向和原地转向。
教训：工程决策不能只考虑成本和简洁性，功能需求和技术可行性才是首要约束。

六、实习总结与体会

今天是焊接日——从热风枪手焊 DFN 到插件元件焊接，最终完成第一块驱动板的焊接；同时确定了双电机驱动方案。收获如下：

  • DFN 手焊是"胆大心细"的活：第一次用热风枪焊接 DFN 封装芯片，300~350°C 的温度控制、锡膏用量的把握、焊后检查的技巧——每一个细节都直接影响焊接质量。纸上谈兵再多，不如亲手焊一块板子来得实在。
  • 并联驱动 vs 双板独立：一个关键的架构决策：如果选择了并联方案，卡丁车将失去差速转向能力，这在后期是无法通过软件弥补的——因为硬件上两个电机已经被强制同步。好在方案讨论中及时发现了这个根本性问题。
  • 电解电容的方向是红线：接反会爆裂，这不是可以"试一试"的问题。焊接前的极性确认和焊后的目视检查，是保护板子和自身安全的底线操作。

总而言之，今天完成了第一块板的焊接和双电机方案决策——板子从"空板"变成了"半成品"，离"能转起来"又近了一步。
